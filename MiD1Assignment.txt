UNIT 1 — CLOUD PRINCIPLES & WELL-ARCHITECTED FRAMEWORK          (W: 15%, T: 7)
================================================================================
Syllabus content:
  Fundamental Principles of Cloud Computing; Shared Responsibility Model;
  Understanding Cloud-Native Architecture; Well-Architected Framework
  (Security, Reliability, Performance, Cost, and Sustainability);
  Best Practices for Designing Scalable Cloud Applications.

--------------------------------------------------------------------------------
Q1.  Fundamental Principles of Cloud Computing                      [7 Marks]
--------------------------------------------------------------------------------
(a) Explain the five fundamental principles of cloud computing:
      on-demand self-service, broad network access, resource pooling,
      rapid elasticity, and measured service.                           [4]
(b) With suitable examples, discuss how rapid elasticity and measured
    service help organizations achieve higher throughput and better
    cost control compared to traditional on-premises IT.                [3]

--------------------------------------------------------------------------------
Q2.  Shared Responsibility Model                                    [7 Marks]
--------------------------------------------------------------------------------
(a) What is the Shared Responsibility Model in cloud computing?
    Clearly distinguish what the cloud provider manages versus what
    the customer manages.                                               [3]
(b) Compare customer vs provider responsibilities for IaaS, PaaS, and
    SaaS (or give a clear comparison for at least IaaS vs PaaS, e.g.
    Compute Engine vs App Engine / similar platforms). Give one
    security implication of misunderstanding this model.                [4]

--------------------------------------------------------------------------------
Q3.  Cloud-Native Architecture                                      [7 Marks]
--------------------------------------------------------------------------------
(a) Explain cloud-native architecture. Discuss any four characteristics
    such as microservices, containers, managed services, API-driven
    design, externalized state, or DevOps/CI-CD alignment.              [4]
(b) How does cloud-native design differ from a traditional monolithic
    application deployed on a single server? Mention scalability and
    failure handling in your answer.                                    [3]

--------------------------------------------------------------------------------
Q4.  Well-Architected Framework & Scalable Design                   [7 Marks]
--------------------------------------------------------------------------------
(a) List the pillars of the Well-Architected Framework (Security,
    Reliability, Performance, Cost, and Sustainability). Explain ANY
    THREE pillars with one concrete cloud practice for each.            [4]
(b) State any three best practices for designing scalable cloud
    applications (e.g., horizontal scaling, load balancing, loose
    coupling, health checks, managed services, design for failure).     [3]

UNIT 2 — DISASTER RECOVERY IN CLOUD                             (W: 15%, T: 7)
================================================================================
Syllabus content:
  Concepts of Backup & Disaster Recovery; High Availability & Fault
  Tolerance in Cloud; Disaster Recovery Strategies (RPO & RTO
  Considerations); Cloud-Based Backup Solutions & Snapshots;
  Case Studies on Disaster Recovery Implementation.

--------------------------------------------------------------------------------
Q1.  Concepts of Backup & Disaster Recovery                         [7 Marks]
--------------------------------------------------------------------------------
(a) Define the terms: Backup, Disaster, Disaster Recovery (DR), and
    Business Continuity.                                                [3]
(b) “Having backups is not the same as having a disaster recovery plan.”
    Justify this statement. In your answer, mention runbooks, restore
    testing, and infrastructure rebuild / failover.                     [4]

--------------------------------------------------------------------------------
Q2.  High Availability, Fault Tolerance & SPOF                      [7 Marks]
--------------------------------------------------------------------------------
(a) Differentiate High Availability (HA) and Fault Tolerance with clear
    definitions and one cloud example for each.                         [3]
(b) What is a Single Point of Failure (SPOF)? Describe how redundancy,
    load balancers, health checks, and multi-zone deployment reduce
    SPOF risk for a cloud web application.                              [4]

--------------------------------------------------------------------------------
Q3.  RPO, RTO & Disaster Recovery Strategies                        [7 Marks]
--------------------------------------------------------------------------------
(a) Define Recovery Time Objective (RTO) and Recovery Point Objective
    (RPO) with one practical example of each.                           [3]
(b) Explain the DR strategy ladder from cold to hot:
      Backup & Restore → Pilot Light → Warm Standby → Hot Standby →
      Multi-site Active-Active.
    For any two strategies, comment on relative cost and typical RTO/RPO. [4]

--------------------------------------------------------------------------------
Q4.  Cloud Backups, Snapshots & Case Study                          [7 Marks]
--------------------------------------------------------------------------------
(a) Describe cloud-based backup / recovery mechanisms you have studied
    (e.g., VM/disk snapshots, object storage versioning / multi-region
    buckets, managed database automated backups & point-in-time
    recovery). State why restore testing is essential.                  [4]
(b) Discuss ANY ONE case study on disaster recovery implementation
    (e.g., e-commerce multi-region design, payments-style system,
    campus exam portal on result day, or a known industry backup
    failure lesson). Map the problem to a suitable DR tier and justify
    using RTO/RPO thinking.                                             [3]

UNIT 3 — INTRODUCTION TO INFRASTRUCTURE AS CODE (IaC)           (W: 20%, T: 10)
================================================================================
Syllabus content:
  Overview of Infrastructure as Code (IaC); Tools for IaC: Terraform,
  AWS CloudFormation, Ansible; Configuration Management & Automation;
  Writing & Managing Infrastructure as Code; Best Practices for
  Versioning & Scaling Infrastructure.

--------------------------------------------------------------------------------
Q1.  Overview of Infrastructure as Code                             [7 Marks]
--------------------------------------------------------------------------------
(a) Define Infrastructure as Code (IaC). Explain why managing
    infrastructure through code is preferred over only manual
    console / GUI provisioning.                                         [3]
(b) State four benefits of IaC (e.g., repeatability, version control,
    automation, reduced human error, documentation, faster DR rebuild).
    Give one real-world scenario where IaC is especially valuable.      [4]

--------------------------------------------------------------------------------
Q2.  IaC Tools — Terraform, CloudFormation & Ansible                [7 Marks]
--------------------------------------------------------------------------------
(a) Compare Terraform, AWS CloudFormation, and Ansible under any four
    of these heads: purpose, language/format, cloud support
    (multi-cloud vs AWS-only), and declarative vs procedural style.     [4]
(b) With a short conceptual example, show how Terraform (or similar)
    can define a simple cloud resource such as a firewall rule, VM,
    or storage bucket.                                                  [3]

--------------------------------------------------------------------------------
Q3.  Configuration Management & Automation                          [7 Marks]
--------------------------------------------------------------------------------
(a) What is configuration management in cloud / DevOps? Differentiate
    provisioning infrastructure (creating VMs, networks) from
    configuring software on those servers.                              [3]
(b) Explain how a tool such as Ansible (playbooks) or startup scripts /
    process managers automate server setup. List typical steps
    (install packages, deploy app code, configure services, run as
    non-root user).                                                     [4]

--------------------------------------------------------------------------------
Q4.  Writing, Versioning & Scaling Infrastructure as Code           [7 Marks]
--------------------------------------------------------------------------------
(a) Describe a recommended workflow for writing and managing IaC:
      write code → version control (Git) → review → plan → apply → verify.
    Why is “plan before apply” important?                               [3]
(b) Explain best practices for versioning and scaling infrastructure
    with IaC (any four): Git tagging, immutable templates, separate
    environments (dev/staging/prod), remote state & locking,
    parameterizing instance counts / autoscaling, never committing
    secrets.                                                            [4]
